Handling the Mailto Function Issue

I had to find a workaround since my Chromium-based browser wouldn't allow me to open the mail client directly. Fortunately, I had encountered this issue before, so I followed these steps:

  1. Disable AdBlock (if enabled).
  2. Go to chrome://settings/handlers and enable "Allow sites to ask to handle protocols."
  3. Open the browser console and type: allow pasting
  4. Copy and paste the following command:
navigator.registerProtocolHandler('mailto', 'https://mail.google.com/mail/?extsrc=...%s', 'Gmail')
  1. Click Allow on the popup that appears.

Observations on User Stories & Design

I found the user stories easy to follow and didn't need to make many assumptions apart from the font size and family. However, I did notice a small typo—the hex code for the pink color is missing an "e" at the beginning. Also, the contrast between gray/black text and the pink background made readability a bit challenging.

To improve accessibility, I adjusted the placeholder color on error states to white, which I believe makes it more user-friendly.

Reflections on the Learning Process and AI

The test cases were incredibly helpful. Having them before starting the project allowed me to visualize the end goal more clearly. I initially created a form that didn't pass, but step by step, I adjusted it until it met all the requirements.

I also made use of generative AI, which has been a game-changer. Although I was skeptical at first, I've come to appreciate how useful it is. In the past, I would have multiple tabs open—MDN, W3C, Stack Overflow, Reddit—going through a lot of trial and error to find the right solution. AI doesn't always provide perfect answers, but it offers great ideas and significantly speeds up the learning process by explaining concepts without the need for extensive research.

That said, AI has its shortcomings. It can encourage laziness—people might copy and paste solutions without fully understanding them, and in many cases, those solutions aren't entirely correct. I've encountered situations where AI-generated code relied on outdated package documentation, leading to errors. So while it's a fantastic tool, it's essential to use it with a critical mindset.

I regularly use ChatGPT, Gemini, Codium, and GitHub Copilot, and I think they all have their strengths. However, I'm not particularly fond of Copilot's inline edits—I don't like having my code rewritten on the fly. I prefer the chat-based approach, where I can review suggestions and decide how to implement them myself. Maybe I just haven't gotten used to Copilot's workflow yet, but for now, that's my stance on it.

Key Takeaways from This Project

I've realized that every project, no matter how simple, always offers something new to learn. With this project, I gained a much deeper understanding of regex—I had used it before for basic form validation, but I had never really explored its full potential. Now, I can see just how powerful and versatile it is.

I also had no prior knowledge of the Luhn algorithm, so learning about it was a fascinating experience. It's always exciting to come across new concepts that expand my problem-solving toolkit!
